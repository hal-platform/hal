Hal - Version 2.8

What is Hal?

Hal is the deployment system we use to deploy many of our client-facing websites, including MyQL, Rocket Mortgage, QLMS, ORMS, Agent Insight, QuickenLoans.com and more!

Changelog

New Features

  • Users can now favorite applications
    Applications can now be starred from the application page and they will be stickied to the top of the dashboard and app list pages.
  • Add ability to automatically deploy successful builds
    Users can now optionally select deployment targets when starting a build. If the build is successful it will be automatically deployed.

    This is optional, and users can still choose to manually deploy after building.
  • Add support for using multiple docker containers
    In your app configuration you can specify containers by prefixing them to the command. If no command is specified, the default or globally specified container will be used. Apps can use as many different containers as they like, in anyorder.

    In the following example, the app-wide default container is PHP 5.6, but the Node 5.3 container is used when compiling frontend dependencies. 
    system: docker:php5.6
build:
    - 'bin/deploy --backend'
    - 'docker:node5.3 bin/deploy --frontend'
  • Add API documentation
    API documentation is linked from the Help page or available here: /docs/api

User Experience (UX) Enhancements

  • Automatically select an environment for building from user preferences
    In most situations users will never need to select their desired environment when building, as Hal remembers which environment they were last viewing.
  • Add more descriptive event messages for build commands
    Build and push commands as well as docker image names will now be listed in event messages, rather than context data. This will make this information easier to find so users do not have to drill down into logs as often.

Bugs

  • Fix unresponsive build menu in Chrome
    In some versions of Chrome the last few items listed in the build menu (which contains branches, tags, and pull requests) could not be selected. This should now be fixed.

Technical

  • Dependencies

    The following components MUST go with this release:

    • Hal 2.8.0
    • Hal Agent 2.6.0
    • Hal Core 2.9.0
    • Kraken Core 2.9.0
  • Hal has been updated to use MCP Panthor 2.3. The open source version of our PHP microframework.
  • Hal Core has been split into Hal Core and Kraken Core to better isolate Kraken components and dependentencies from Hal.
