HAL 9000 - Version 2.6.1

Greetings! The web core team has deployed an update to HAL 9000. This is version 2.6.1.

What is HAL 9000?

HAL 9000 is the deployment system we use to deploy many of our client-facing websites, including MyQL, Rocket Mortgage, QLMS, ORMS, Agent Insight, QuickenLoans.com and more!

Changelog

Improvements

  • Kraken Runtime Configuration now supports QKS.

    Kraken is now fully functional and prod-ready.

Bugs

  • Fix doctrine cache clearing from frontend

Changes

  • Cookies are now encrypted with libsodium. This will wipe all existing settings.

API Changes

  • Application
    • title changed to name.
    • eb-name changed to eb_name.
    • url removed.
    • github-user removed.
    • github-repository removed.
    • page link added.
    • status_page link added.
    • github_page link added.
  • Build
    • reference is now a string, instead of object containing text and url.
    • commit is now a string, instead of object containing text and url.
    • url removed.
    • page link added.
    • github_reference_page link added.
    • github_commit_page link added.
    • start_push_page link added if a build can be pushed.
  • Push
    • url removed.
    • page link added.
    • deployment link may be missing, rather than null.

Technical

  • mcrypt is no longer required. libsodium replaced mcrypt functionality.
  • Dependencies

    The following components MUST go with this release:

    • HAL 2.6.1
    • HAL Core 2.7.1
    • HAL Agent 2.4.1

Need Help?

If you find any bugs or want to request a feature, drop us a line in the hipchat room [Application] HAL 9000, or send us an email.
