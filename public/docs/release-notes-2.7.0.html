HAL 9000 - Version 2.7

What is HAL 9000?

HAL 9000 is the deployment system we use to deploy many of our client-facing websites, including MyQL, Rocket Mortgage, QLMS, ORMS, Agent Insight, QuickenLoans.com and more!

Changelog

New Features

  • AWS CodeDeploy is now supported.

User Experience (UX) Enhancements

  • Deployments are now categorized using the user's selected deployment pools on the Start Push page.
  • Configuration deployed with Kraken is now batch encrypted.
    Kraken deployments should be slightly faster, and clients can now use batch decryption to significantly decrease the load placed on QKS.
  • Build refs are limited to 3 per row on the Start Build page, and PRs owned by the current user are highlighted.

Bugs

  • Miscellaneous UI inconsistencies were fixed.
  • Elastic Beanstalk deployments broken during v2 to v3 SDK upgrade have been fixed.
  • Missing temp folder would cause jobs to fail after an agent is restarted.
  • Fix Add Application page broken by GitHub organizations exceeding 100.

Technical

  • mcp-crypto updated to 2.1.3
  • twig updated to 1.22
  • Dependencies

    The following components MUST go with this release:

    • HAL 2.7.0
    • HAL Agent 2.5.0
    • HAL Core 2.8.3
