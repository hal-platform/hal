Hal - Version 2.5

What is Hal?

Hal is the deployment system we use to deploy many of our client-facing websites, including MyQL, Rocket Mortgage, QLMS, ORMS, Agent Insight, QuickenLoans.com and more!

Changelog

Developer Experience (DX) Enhancements

  • All builds now run within Docker containers.

    The following containers are currently available at the time of this release.

    • Elixir 1.0.4
    • Erlang 17.5
    • HHVM 3.7
    • PHP 5.4
    • PHP 5.5
    • PHP 5.6
    • PHP 7.0
    • Legacy (Default)

    All PHP containers have Node.js installed. The legacy container has Ruby 1.9.3 as well.

    See Deployment scripting guide for more details.

  • The following projects have been tested and verified to build successfully in the legacy container.
    • agent-insight
    • myql-common
    • myql-origination
    • myql-servicing
    • phoenix
    • qlms
    • qlms-html
    • qlms-portal
    • Q4
    • rocket-web

Technical

  • Hal Agent now runs on a new dedicated server.

    It has more power!

    It is more secure!

    However, currently there is only one. Previously Hal jobs ran on two separate servers.

  • Added docker:status command to get system information and docker status information.
    This data is pushed to redis so it may be read from the frontend.
  • Added server:connections command to test ability to connect to deployment servers.
    This data is pushed to redis so it may be read from the frontend.
  • Added docker:refresh command to push dockerfiles to build server.
    An audit log is recorded when this command is run.
