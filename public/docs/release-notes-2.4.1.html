HAL 9000 - Version 2.4.1

What is HAL 9000?

HAL 9000 is the deployment system we use to deploy many of our client-facing websites, including MyQL, Rocket Mortgage, QLMS, ORMS, Agent Insight, QuickenLoans.com and more!

New Features

  • Add daily stats to login page.

Changelog

User Experience (UX) Enhancements

  • All users can now add groups or repositories.

    Previously, only administrators could do this.

  • Restore Rollback page for all users.

    The rollback page has been changed to a complete deployment history and lists all previous pushes for a deployment. It is available to all users by clicking the rollback icon on the titlebar of a push status box.

  • Add ability to view all pushes and builds for a particular date.

    Available through the administration dashboard, administrators can view all pushes or builds that occured on a particular date.

  • Relative times are now handled by javascript.

    Relative times will be updated on-the-fly, so times displayed in the UI will be more accurate. This is will also result in correct times for users not in EST.

Bugs

  • HAL can now deploy repositories of any size.

    Previously, HAL was limited to what it could download into memory before writing to disk.

Technical

  • HAL now uses YAML configuration for Doctrine mapping, rather than annotations.
  • Dependencies

    The following components MUST go with this release:

    • HAL 2.4.1
    • HAL Core 2.6
